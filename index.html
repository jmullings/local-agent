Local-First Agent Infrastructure

An agent that never leaves your machine.

Local Agent mirrors your project into Docker, attaches a tool-calling LLM over MCP, and lets it write, run, and verify its own work, scanned and audited the whole way, without your code touching the cloud.

What's live today

End-to-end control. Fully local.

Everything below runs inside the extension right now — no waitlist, no beta flag.

🐳
Shipped

Mirror & manage

Point Local Agent at a folder and it generates a Dockerfile, builds the image, and runs your project in a container with the agent attached. Already have infrastructure running? Attach the agent to it directly over MCP instead.

🔌
Shipped

MCP tool access

Every tool the agent can call is toggled on by you, per session, from the sidebar. Nothing is granted by default, and nothing you haven't enabled is reachable.

⚙️
Shipped

Autonomous loop

Hand it an objective — write tests for this file and run them until they pass — and watch it read, write, run, and iterate end-to-end inside the same chat thread.

🛡️
Shipped

Safeguard

A baseline OWASP ZAP scan that can only ever target the container Local Agent just built for you. There's no field to point it anywhere else.

🔒
Shipped

Encrypted mirroring

Mirrored projects can sit behind a 6-digit PIN, set before the container is even built, that encrypts access to the local copy.

code --install-extension local-agent-0.0.1.vsix
